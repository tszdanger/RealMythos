# Stage 4 Trace Scaffold

Stage 4 will explore multiple scaffold designs for collecting richer security reasoning traces.

The goal is to support future dataset expansion with trace records that are more diverse, more inspectable, and easier to evaluate across different reasoning workflows.

## Planned Directions

- Static-analysis-assisted scaffold
- Dynamic execution scaffold
- Patch-diff-aware internal scaffold with patch-unaware public output
- Multi-reviewer trace validation scaffold
- Human-in-the-loop validation scaffold
- Environment-grounded PoC scaffold
- Failure-analysis scaffold

## Expected Outputs

- Scaffold interface specification
- Trace schema
- Collection runner design
- Quality evaluation protocol
- Safety review process
- Example scaffold implementations
